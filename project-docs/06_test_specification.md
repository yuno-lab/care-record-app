# テスト仕様書

## 1. 対象者名管理
### TS-N-001 名前追加
- 前提: names.html を開く
- 手順: 新しい名前を入力して追加
- 期待結果: 一覧に追加される

### TS-N-002 重複追加防止
- 前提: 同じ名前が1件登録済み
- 手順: 同じ名前を追加
- 期待結果: エラーダイアログが表示され、追加されない

## 2. 記録保存
### TS-R-001 正常保存
- 前提: 対象者名が登録済み
- 手順: 日付、対象者名、任意項目を入力し保存
- 期待結果: 記録が保存済み記録一覧に表示される

### TS-R-002 重複保存防止
- 前提: 同一 personId + date の記録が既に存在
- 手順: 同じ personId + date で保存
- 期待結果: 保存不可、エラーダイアログ表示

## 3. 記録編集
### TS-E-001 編集開始
- 手順: 保存済み記録の編集ボタンを押す
- 期待結果: 編集フォームが表示される

### TS-E-002 編集保存
- 手順: 内容変更後に保存
- 期待結果: 確認ダイアログ表示後、一覧に更新反映

### TS-E-003 編集キャンセル
- 手順: キャンセル押下
- 期待結果: 確認ダイアログ表示後、編集状態が解除される

### TS-E-004 編集時重複防止
- 前提: 他記録に同一 personId + date が存在
- 手順: 編集で同一キーに変更して保存
- 期待結果: 保存不可、エラーダイアログ表示

## 4. JSON 入出力
### TS-J-001 JSON エクスポート
- 手順: JSONバックアップを実行
- 期待結果: names と records を含む JSON がダウンロードされる

### TS-J-002 JSON インポート
- 前提: JSON ファイル準備済み
- 手順: インポートを実行
- 期待結果: names はマージ、records は personId + date 一致時に上書き

## 5. CSV 出力
### TS-C-001 CSV エクスポート
- 手順: 表示中の記録をCSVエクスポート
- 期待結果: CSV ダウンロード、Excelで読みやすい

## 6. 印刷
### TS-P-001 印刷表示
- 手順: 印刷ボタン押下
- 期待結果: フォームや操作ボタンが非表示、記録のみ表示

## 7. 絞り込み
### TS-F-001 対象者絞り込み
- 手順: 対象者を選んで絞り込み
- 期待結果: 該当記録のみ表示

### TS-F-002 日付絞り込み
- 手順: 開始日 / 終了日を指定
- 期待結果: 期間内記録のみ表示

## 8. レスポンシブ
### TS-S-001 スマホ表示
- 手順: 幅760px以下で確認
- 期待結果: ナビ、フォーム、ボタンが縦並びで操作しやすい
